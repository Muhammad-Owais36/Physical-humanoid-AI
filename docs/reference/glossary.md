# Glossary of Terms

This glossary defines key terms used throughout the Physical AI & Humanoid Robotics documentation.

## A

### Action (ROS 2)
A communication pattern in ROS 2 for long-running tasks with feedback and goal management.

### Admittance Control
A control strategy that relates applied forces to resulting velocities, opposite to impedance control.

### AI-Robot Brain
The cognitive component of the Physical AI system, typically implemented using NVIDIA Isaac for perception and decision-making.

## B

### Balance Control
Control algorithms that maintain a humanoid robot's stability during static and dynamic activities.

### Behavior Tree
A hierarchical structure used for organizing robot behaviors and decision-making processes.

## C

### Cartesian Space
The 3D space defined by X, Y, Z coordinates, as opposed to joint space.

### Control Space
The mathematical space in which control algorithms operate, such as joint space or Cartesian space.

### Coordinate Frame
A reference system used to define positions and orientations in 3D space.

## D

### Degrees of Freedom (DOF)
The number of independent movements a mechanical system can make.

### Digital Twin
A virtual representation of a physical system used for simulation, testing, and optimization.

### Dynamic Model
A mathematical representation of how forces and torques affect a robot's motion.

## E

### End Effector
The tool or grasping mechanism at the end of a robot arm.

### Environment Mapping
Creating a representation of the robot's surroundings for navigation and interaction.

## F

### Forward Kinematics
The process of calculating end-effector position from joint angles.

### Force Control
Control strategies that regulate the forces applied by the robot to its environment.

## G

### Gazebo
An open-source physics simulation environment commonly used in robotics.

### Generalized Coordinates
A set of parameters that uniquely define the configuration of a mechanical system.

## H

### Hardware Abstraction Layer (HAL)
Software layer that provides a consistent interface to different hardware implementations.

### Humanoid Robot
A robot with human-like form and capabilities, designed to operate in human environments.

## I

### Impedance Control
A control strategy that relates position errors to resulting forces, commonly used for compliant interaction.

### Inverse Dynamics
The calculation of forces and torques required to achieve a desired motion.

### Inverse Kinematics
The process of calculating joint angles needed to achieve a desired end-effector position.

### Isaac Sim
NVIDIA's simulation environment for robotics, providing photorealistic rendering and physics simulation.

## J

### Jacobian Matrix
A mathematical construct that relates joint velocities to end-effector velocities.

### Joint Space
The space defined by joint angles, as opposed to Cartesian space.

## K

### Kinematic Chain
A series of rigid bodies connected by joints that transmit motion.

### Kinematics
The study of motion without considering the forces that cause it.

## L

### Locomotion
The ability to move from one place to another.

### Low-Level Controller
A controller that directly interfaces with hardware, typically running at high frequencies.

## M

### Middleware
Software that provides common services and capabilities to applications beyond what's offered by the operating system.

### Motion Planning
The process of determining a sequence of movements to achieve a goal while avoiding obstacles.

## N

### Navigation2
The navigation stack in ROS 2 for autonomous robot navigation.

### Node (ROS 2)
A process that performs computation in the ROS 2 system.

## P

### Perception Pipeline
A sequence of algorithms that process sensor data to understand the environment.

### Point Cloud
A set of data points in 3D space, typically generated by 3D sensors.

### Proportional-Integral-Derivative (PID) Controller
A control loop feedback mechanism widely used in robotics.

## R

### Robot Operating System 2 (ROS 2)
Middleware for robotics applications providing hardware abstraction, device drivers, and communication.

### ROS 2
The second generation of the Robot Operating System, designed for production robotics.

### ROS 2 Node
An executable that uses ROS 2 client libraries to communicate with other nodes.

### ROS 2 Package
A container for related files and code in the ROS 2 system.

## S

### Sensor Fusion
The process of combining data from multiple sensors to achieve better accuracy and reliability.

### Service (ROS 2)
A communication pattern in ROS 2 for request-response interactions.

### Simulation
The representation of a real-world system in a virtual environment.

## T

### Task Space
The space in which the robot's task is naturally expressed, often Cartesian space for manipulation tasks.

### Trajectory Planning
The process of creating smooth, feasible paths for robot motion.

## U

### Unity
A game engine used for creating high-fidelity simulations and visualizations.

### URDF (Unified Robot Description Format)
An XML format for representing robot models in ROS.

## V

### Velocity Control
Control strategies that regulate the speed of robot motion.

### Vision-Language-Action (VLA)
Integration of visual perception, language understanding, and physical action capabilities.

### Visual Servoing
Control strategy that uses visual feedback to guide robot motion.

## W

### Workspace
The volume of space that a robot can reach with its end effector.

### Whole-Body Control
Control strategies that consider the entire robot system simultaneously.

## Z

### Zero-Moment Point (ZMP)
A concept in bipedal robotics used for balance and gait stability analysis.
